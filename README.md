Предварительные условия:

На вашей операционной системе должен быть установлен Docker Desktop если вы пользователь Windows или Mac OS, устанавливать не требуется если вы напрямую работаете с Linux (Ubuntu, Debian и так далее)

Инструкция по запуску проекта:

Клонируем github репозиторий в любое удобное для вас место
(git clone https://github.com/kirilloid173/blogs_fullstack_docker.git)
Переходим к папке проекта

Создаём .env файл в корне проекта

Внутри .env файла прописываем значения записанные в .env.example, с записью переменных записанных в ней

Записываем значение DB_ROOT_PASSWORD которая будет использоваться для проверки работоспособности базы
данных (MySQL)

Создаём .env файл в папке Laravel, полностью копируем содержимое .env.example с папки laravel в созданный нами .env

Открываем командую строку вашей операционной системы и переходим к папке проекта

Запускаем команду docker compose build для создания билда php образа

Запускаем команду docker compose up -d для инициализации запуска проекта

Запускаем команду docker compose exec php sh для того чтобы зайти внутрь нашего проекта, это необходимо для запуска Laravel

Запускаем команду composer update для обновления пакетов

Запускаем команду composer install для установки необходимых пакетов (чтобы точно убедиться в корректной установке)

Запускаем команду npm install для установки Node/Vite пакетов, необходимых для работы Laravel

Запускаем команду php artisan migrate:refresh --seed для пересоздания миграций и данных в базе данных, а также для добавления записей в БД (seeders)

Запускаем последнюю команду composer run dev, для запуска Laravel Backend API сервиса

Готово, теперь можно открывать локальную страницу в вашей операционной системе http://localhost:80/
